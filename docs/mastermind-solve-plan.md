## 1. 规则回顾(玩家可见)

| 反馈 | 含义 |
| --- | --- |
| **位准**(黑钉) | 某枚颜色**位置也对** |
| **色准**(白钉) | 颜色对、**位置不对**;已算位准的不再算色准 |
| 重复色 | 仅「困难」秘密码与猜测可重复;简单/普通禁止 |

计分顺序:**先扫位准,再对剩余做多重集合匹配**——与经典 Mastermind 一致(已有单测锁定)。

## 2. 推荐破译策略(给玩家 / 给引导文案)

### 2.1 核心原则

1. **每步最大化信息量**:优先覆盖未试过的颜色与位置,少做「几乎重复上一次」的猜测。  
2. **分离「有哪些色」与「在哪」**:前几手用互不重叠的调色盘摸清色集合,再用置换猜位置。  
3. **用历史做排除**:若某次 `位准+色准 = 0`,这手所有颜色可暂时视为「不在密码里」(无重复档绝对成立;可重复档需更谨慎)。  
4. **困难档防坑**:同一颜色可出现多次——色准/位准可能来自重复钉,不要按「六色各一」心智模型硬套。

### 2.2 分难度开局建议

| 难度 | 建议开局 | 理由 |
| --- | --- | --- |
| 简单(3/5/无重复) | 任意 3 个不同色 | 空间小,2–4 手内常可锁死 |
| 普通(4/6/无重复) | 第 1 手 4 个不同色;第 2 手换上剩余 2 色 + 置换 | 经典六色四位;两手可摸清「在盘里的颜色」 |
| 困难(5/8/可重复) | 第 1 手 5 个不同色;随后用「双同色试探」确认是否有重复 | 先估色集合,再测重复与位置 |

### 2.3 一手解读速查

假设密码长度 4:

- `4 位准` → 通关  
- `0 位准 · 4 色准` → 颜色全对、位置全错 → 下一手做轮换/对换  
- `2 位准 · 0 色准` → 两个位置钉死,另外两色不在剩余槽的「错位候选」里(或根本不在码中)  
- `0 · 0` → 本手颜色整组排除(无重复档)

### 2.4 可做成局内「提示」(尚未实现)

- 卡住时给**一条**策略句(非直接给答案),例如「试试把上一手的前两色对调」。  
- 简单档可提供「最优下一步」练习模式(求解器),默认关闭以免掉可玩性。

## 3. 求解器方向(工程可选)

若要做「提示 / 自动演示 / 难度校验」:

1. **候选集**:开局枚举全部合法密码(普通档 \(P(6,4)=360\);困难档 \(8^5=32768\),可接受)。  
2. **每步过滤**:对每个候选跑与玩家相同的 `evaluateGuess`,只保留反馈一致者。  
3. **选下一步**:在剩余候选上选最小化最坏情况分支的猜测(Knuth 风格),或随机合法猜测保手感。  
4. **实现落点**:纯函数放 `src/games/mastermind/solver.ts`,UI 只调接口;单测用固定秘密码锁死「≤N 步必中」。
