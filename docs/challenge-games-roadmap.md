# 挑战局品类路线图:多款 Mastermind 同类益智

> 日期:2026-07-23  
> 目的:**不是**单局破译攻略,而是「如何在一局里持续做多款同型益智挑战」。  
> 现状:M13《密码破译》已落地,作为品类样板。

---

## 1. 「同类型」定义(选型过滤器)

后续挑战局必须同时满足:

| 条件 | 说明 |
| --- | --- |
| 一局制 | 5–15 分钟,胜负明确 |
| 可定难度 | ≥3 档(盘面/长度/步数/雷密度等) |
| 技能表达 | 靠推理/规划,不是剧情点选 |
| 反馈可测 | 规则纯函数,Vitest 可锁 |
| 工程边界 | 只走 `src/challenges/` + `src/games/{id}/`,**禁止** import `engine/` / 故事 `content/` |
| 静态可部署 | 无服务端 API(Pages 友好) |
| UI 一致 | 浅色暖纸;主页卡 → 介绍 → `/play` |

**刻意不做(本路线图外):**叙事文字局、3D、强联网对战、重度卡组。

**与密码破译的「同类」核心:**  
玩家做一次可验证的尝试 → 系统给结构化反馈 → 缩小假设空间 → 通关/失败。

---

## 2. 工程骨架(先抽壳,再加游戏)

当前 Mastermind 把「品类壳」和「具体规则」耦在同一套页面里。多款之前建议先做一次**薄抽取**(可作为 M14):

```text
src/challenges/
  types.ts              # ChallengeMeta(+可选 genre 标签)
  registry.ts           # list/get
  {id}/meta.ts          # 每局文案+封面

src/games/
  {id}/rules.ts         # 纯规则+单测
  {id}/...

src/components/challenges/
  ChallengeShell.tsx    # 浅色页框、回介绍、难度条、本机最佳
  ChallengeIntroLayout  # 与模组介绍同构的介绍页布局

app/challenges/[id]/page.tsx       # 通用介绍(读 registry)
app/challenges/[id]/play/page.tsx  # 按 id 动态挂载 Game 组件
```

**原则:**新开一款游戏 = 加 `meta` + `rules` + `XxxGame.tsx` + 封面 + 注册表一行;**不**复制整套路由。

M13 可先保持专用路由;M14 再迁到动态 `[id]`,避免一次大爆。

## 3. 候选池(按推荐开发顺序)

| 序号 | 游戏 | 一句话 | 与 Mastermind 的同类点 | 难度旋钮 | 预估工期 | 建议里程碑 |
| --- | --- | --- | --- | --- | --- | --- |
| 0 | **密码破译** | 色码序列 + 位准/色准 | 样板 | 长度/色数/重复/次数 | 已完成 | **M13** |
| 1 | **海战棋** | 猜对方舰位(Battleship) | 坐标猜测 + 命中/未中/沉舰反馈 | 盘面、舰数、是否允许重击提示 | 4–6 天 | **M15** |
| 2 | **扫雷** | 经典开格推理 | 点击 → 数字线索 → 排除 | 宽高、雷密度、是否标旗 | 3–5 天 | **M16** |
| 3 | **数织** | Nonogram 黑白格 | 线索数组 → 填格收敛 | 网格、线索密度、限时 | 5–7 天 | **M17** |
| 4 | **推箱子·短关** | 3D 非必须,等距即可 | 状态空间搜索(偏空间益智) | 关卡表=难度 | 6–8 天 | **M18**(可选) |

**首发后下一款推荐:海战棋(M15)。**  
理由:反馈模型与 Mastermind 最近(猜→结构化反馈→收束),可复用「历史列表 + 右栏操作 + 本机最佳」交互骨架,又能明显换新鲜感。

备选若更想「大众秒懂」:扫雷(M16)可与海战棋对调。

---

## 4. 里程碑切片(写入开发计划的节奏)

### M14 — 挑战局壳抽取(1–2 天)

- 动态路由 `/challenges/[id]`、`/challenges/[id]/play`
- 共用浅色 Shell、难度选择器、本机最佳存储接口
- Mastermind 迁入壳内,行为回归(单测+E2E)全绿
- 主页仍只展示 `listChallenges()`

### M15 — 海战棋(Battleship)

- `src/games/battleship/rules.ts` TDD(放置校验、开火、沉舰、胜负)
- 简单/普通/困难盘面表
- 介绍页 + 浅色棋盘 UI + Dofun 封面
- E2E:主页→介绍→开炮见反馈

### M16 — 扫雷

- 规则纯函数 + 三档雷密度
- 复用 Shell;棋盘组件可与海战棋分文件,不硬抽过早
- E2E:开一局标旗/点开不死

### M17 — 数织(按带宽)

- 关卡数据驱动(JSON/TS 关卡表)
- 困难档用更大网格

每完成一款:`status: playable` 进主页;未开工只放 `preview` 占位(与故事局一致)。

## 5. 单款游戏交付清单(复制用)

每开新游戏按此勾选:

1. [ ] `rules.ts` + `rules.test.ts`(RED→GREEN)
2. [ ] `challenges/{id}/meta.ts` + registry 注册
3. [ ] Dofun 封面 → `public/challenges/{id}/cover.webp` + `art-briefs.md`
4. [ ] `XxxGame.tsx` 浅色 UI(可定难度、胜负页、本机最佳)
5. [ ] 介绍文案(设定/玩法/难度一览)
6. [ ] E2E 一条主路径
7. [ ] `PROJECT_STATUS` / 开发计划勾选对应 M*

---

## 6. 明确不做的扩展

- 不为「再来一款」去改故事引擎
- 不把挑战局接到 DeepSeek
- 不并行开两个挑战里程碑 PR(与 AGENT 一致:一个里程碑闭环后再开下一个)
